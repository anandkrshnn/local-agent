# Contributing to the Sovereign AI Ecosystem

Thank you for your interest in contributing to the Sovereign AI Ecosystem! We welcome contributions that help advance the state of verifiable, private, and autonomous AI.

## How to Contribute

### Reporting Bugs
- Use the **Bug Report** template in GitHub Issues.
- Provide a clear, concise description and steps to reproduce.

### Suggesting Features
- Use the **Feature Request** template in GitHub Issues.
- Explain why the feature is needed and how it fits the PTV Protocol philosophy.

### Code Contributions
1. Fork the repository.
2. Create a new branch (`feat/your-feature` or `fix/your-fix`).
3. Ensure your code follows the existing style and is well-documented.
4. Add tests if applicable.
5. Submit a Pull Request with a clear description of the changes.

## Standards Track (PTV Protocol)
For contributions to the specification itself (`ptv-protocol-spec`):
- All changes must align with the **IETF Standards Track** requirements.
- Major changes may require a formal RFC-style proposal in the `discussions` area first.

## Licensing
By contributing to this repository, you agree that your contributions will be licensed under the project's existing license (MIT or CC BY 4.0).
